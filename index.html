Welcome

The Social Media Ostracism paradigm is a web-based experimental manipulation of ostracism and social exclusion, designed for research purposes. On this page you can find:

  • Preview of the paradigm (online version)
  • How to use the online version of the paradigm for experimental purposes
  • How to download & customize the study
  • How to install and run the downloaded versions on a personal computer
  • Suggestions on how to run the customized study online

Try it out

To try out the Social Media Ostracism paradigm follow this link: http://ostracism.github.io/preview/index.html?c=1&p=999 (exclusion condition) http://ostracism.github.io/preview/index.html?c=2&p=999 (inclusion condition)

More about the paradigm

Social media ostracism is a type of social exclusion which happens via social media websites, such as Facebook, Twitter, blogging platforms, or forums. In the paradigm, feelings of exclusion are conveyed through the number of "likes" that people receive on their personal profiles during a group intro task. A detailed description of the procedure and report of the manipulation effects can be found in this manuscript: ..

We recommend that you end the paradigm with an online survey, which contains manipulation checks, dependent measures, and/or connection to subsequent tasks. If the entire study online is conducted online and with no interruptions, the participants will be more likely to think that they are interacting with other participants (rather than computer scripts).

The Social Media Ostracism paradigm was inspired by Cyberball - a virtual ball-tossing game. You can find out more about Cyberball and research on ostracism on https://cyberball.wikispaces.com/

Use it in an experiment

To use the paradigm in your own experiment without additional customization, you will only need the following link:

http://ostracism.github.io/preview/index.html?c=XXXX&p=YYYY&qid=ZZZZ

Substitute... with...
XXXX 1 for ostracism condition, 2 for inclusion, 3 for overinclusion
YYYY unique participant number
ZZZZ Encoded* survey ID

e.g. http://ostracism.github.io/preview/index.html?c=1&p=12&qid=http%3A%2F%2Ffppvu.qualtrics.com%2FSE%2F%3FSID%3DSV_a9u9MdnpIRuxctT

To encode your survey link, open http://meyerweb.com/eric/tools/dencoder/ , copy the active link to your survey, paste it, and press "Encode." Then copy the new, encoded link, and paste it in the url (in place of ZZZZ).

Set it up on your own computer or server

Download the XX.zip file and extract the files to your computer (unzip). To run the study, open the file "main.js" in an internet browser (internet connection is not necessary).

By customizing the code, you can make changes in the following parameters:

  • The task instructions
  • Avatar pool from which the participant can choose their own avatar
  • Number of people the participant "interacts" with
  • Profiles (avatars + descriptions) of other group members
  • Number of "likes" received by the participant and other group members
  • Change the word "like" to another word or symbol
To adjust various parameters:
  1. Open the file main.js (from the folder on your computer) in a source code editor (e.g. Notepad++)
  2. Open the Instructions-MainJS document:
  3. The code you see in the editor (main.js) matches the code you see on the right side of the instructions document
  4. Change the text in the main.js file on your computer (using the editor), according to the instructions document
  5. Save the changes made to the main.js document
To change the group member profiles (descriptions and avatars):
  1. Open the file tmpl_others.js (from the folder on your computer) in a source code editor (e.g. Notepad++)
  2. Open the Instructions-OthersJS document:
  3. Change the text in the index.html file on your computer (using the editor), according to the instructions document

  4. Save the changes made to the index.html document

    To change the study instructions:

  5. Open the file index.html (from the folder on your computer) in a source code editor (e.g. Notepad++)

  6. Open the Instructions-IndexHTML document:

  7. Change the text in the index.html file on your computer (using the editor), according to the instructions document

  8. Save the changes made to the index.html document
